# 功能关键词定义示例

这是 `analyze_kano.py` 所需的 `features.json` 格式示例。将此文件作为模板，替换为你实际项目的功能和关键词。

## JSON 格式

```json
{
  "功能名A": {
    "desc": "一句话描述该功能",
    "keywords": ["关键词1", "关键词2", "关键词3", "用户口语化表达..."]
  },
  "功能名B": {
    "desc": "一句话描述该功能",
    "keywords": ["关键词1", "关键词2", "..."]
  }
}
```

## 关键词设计原则

1. **口语化优先**：访谈是口语，覆盖不同表达方式（如"跌倒""摔跤""摔倒"）
2. **数量控制在 10-20 个**：太少遗漏证据，太多（尤其宽泛词）导致误命中
3. **避免过宽词**：单独的"好""用""有""需要"等误命中率极高
4. **领域特有词优先**：专有名词命中准确率高（如"血氧""挂号""助眠"）
5. **功能边界清晰**：同一关键词只归属一个功能，避免重复计数
